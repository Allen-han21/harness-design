# Adapters

이 디렉터리는 공통 하네스 철학을 각 도구에 어떻게 연결할지 정리하는 곳이다.

핵심 원칙:

- `principles/`는 도구에 무관한 공통 철학을 담는다
- `adapters/`는 각 도구의 activation 방식과 적용 차이를 담는다
- 도구별 문서는 철학을 다시 정의하지 않는다
- 도구별 문서는 "어떻게 읽히게 하고, 어떻게 적용할지"에 집중한다

## 분리 기준

### `principles/`에 들어갈 것

- 문서 수명주기
- 승격 규칙
- 로컬 작업 공간 원칙
- 문서 언어 정책
- 평가와 검증의 일반 원칙
- 사람 개입 기준

### `adapters/`에 들어갈 것

- 도구가 무엇을 먼저 읽는지
- 어떤 instruction 파일이 진입점인지
- 어떤 문서 길이와 형식이 안정적인지
- 자주 발생하는 실패 패턴
- 프로젝트에 얇게 적용하는 방법

## 운영 규칙

- 공통 원칙은 한 곳에서만 정의한다
- 도구별 문서는 공통 원칙을 참조하고, 도구 특화 차이만 덧붙인다
- 프로젝트에 실제 적용할 때는 adapter 문서를 그대로 복사하지 말고 요약해서 넣는다
- 도구별 차이가 실제로 의미 있을 때만 adapter를 확장한다
